# 싱크 미팅 — 레거시 연동 방안 논의

- **일시**: 2026-01-15 (목) 14:00–15:10
- **장소**: 온라인 회의
- **참석자**: 기획리드, 백엔드리드, 프론트엔드리드

---

## 1. 미팅 목적

튜터 제외(차단) 기능을 기존 MyPage와 어떻게 연동할지 기술적 방향을 잡기 위한 싱크 미팅.

## 2. 현황 공유

### MyPage 현재 구조

- MyPage는 Vue.js 2 기반 레거시 SPA
- 수업 예약, 수업 이력, 수강권 관리 등 핵심 기능이 모여 있음
- 최근 신규 기능들은 React로 개발 중이며, MyPage 내 iframe 또는 별도 라우팅으로 연결하는 패턴 사용

**프론트엔드리드**: "MyPage에 직접 Vue 컴포넌트를 추가하는 건 유지보수 비용이 너무 커요. 최근에 수강권 변경 페이지도 React로 따로 만들고 MyPage에서 링크로 연결했거든요. 튜터 차단 관리도 같은 패턴으로 가는 게 맞을 것 같습니다."

**기획리드**: "그러면 MyPage 좌측 메뉴에 '튜터 관리' 같은 항목을 하나 추가하고, 클릭하면 React 페이지로 이동하는 방식?"

**프론트엔드리드**: "네, 정확히 그 방식이요. 인증 토큰은 URL 파라미터로 넘기고, React 앱에서 API 호출해서 차단 목록을 보여주면 됩니다."

### 수업 후 평가 팝업 관련

**기획리드**: "평가 팝업은 어디서 뜨는 게 자연스러울까요? ClassBoard에서 수업 끝나고 나오면 바로?"

**백엔드리드**: "ClassBoard 쪽에서 직접 팝업을 띄우는 건 좀 어려워요. ClassBoard는 별도 도메인이라 세션 공유가 안 되거든요. 수업 종료 후 EduTalk 메인으로 돌아왔을 때 띄우는 게 현실적입니다."

**프론트엔드리드**: "예약 탭으로 돌아오는 시점에 미평가 수업이 있으면 팝업을 띄우는 로직으로 가면 될 것 같아요. 하루에 한 번만 노출하는 조건도 걸고요."

**기획리드**: "좋아요. 그러면 ClassBoard 나와서 → 예약 탭 진입 → 미평가 수업 체크 → 팝업 노출 순서네요."

### 과거 수업 이력에서의 차단

**기획리드**: "수업 이력 페이지에서도 차단할 수 있으면 좋겠다는 의견이 있었는데요. 과거에 불쾌했던 튜터를 나중에 떠올리고 차단하고 싶은 케이스."

**프론트엔드리드**: "수업 이력 리스트에 각 항목마다 '이 튜터 차단하기' 버튼을 넣는 건 UI가 좀 복잡해질 수 있어요. 수업 상세 페이지 안에 넣는 게 나을 것 같습니다."

**백엔드리드**: "API 쪽은 어차피 튜터 ID로 차단 추가하는 거라 진입점이 어디든 상관없어요. 다만 수업 이력 상세에서 차단하면 바로 확인 팝업을 띄워서 실수 방지하는 게 좋겠습니다."

**기획리드**: "그러면 진입점 정리하면: (1) 수업 후 평가에서 낮은 점수 줄 때 차단 제안, (2) 수업 이력 상세에서 직접 차단, (3) 차단 관리 페이지에서 추가/해제. 이 3개로 가죠."

## 3. 차단 관리 페이지 위치

**기획리드**: "차단 관리 페이지를 MyPage 메뉴에 넣을 건데, 메뉴명은 어떻게 하죠?"

**프론트엔드리드**: "기존 메뉴가 '내 정보', '수강권', '수업 이력', '결제 내역'이니까... '수업 설정'이라는 카테고리를 만들고 그 안에 '튜터 관리'를 넣으면 어떨까요?"

**기획리드**: "'수업 설정 > 튜터 관리'... 나쁘지 않네요. 나중에 선호 튜터 기능이 추가되면 같은 카테고리에 넣을 수 있고."

**백엔드리드**: "동의합니다. 확장성도 있고요."
